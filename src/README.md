# Source Directory

## Architecture

This is a single-page static site. No framework, no build step. One `index.html` file with embedded CSS and JS.

### Why single-file?
- GitHub Pages deploys instantly with zero config
- No build pipeline to break
- Any collaborator can edit it
- Performance is inherently good (one HTTP request)
- The site is small enough that splitting files adds complexity without benefit

### When to split
If the site grows to include multiple pages (e.g., separate pages for each team member, a blog, or detailed project documentation), migrate to a static site generator like Astro or 11ty. Until then, keep it simple.

## Content Rendering

All content is inlined from `/data/*.json` at build time. The develop agent (Agent 03) reads the JSON and writes it directly into the HTML. This means:

1. Content updates require re-running the develop agent (or manually editing the inlined data)
2. The HTML is self-contained — no fetch() calls needed at runtime
3. The site works when opened as a local file (no server required)

### Alternative: Runtime fetch
If you prefer dynamic loading (so JSON edits auto-reflect without rebuilding HTML):

```javascript
async function loadData() {
  const [project, team, pubs, positions] = await Promise.all([
    fetch('/data/project.json').then(r => r.json()),
    fetch('/data/team.json').then(r => r.json()),
    fetch('/data/publications.json').then(r => r.json()),
    fetch('/data/positions.json').then(r => r.json())
  ]);
  renderSite({ project, team, pubs, positions });
}
```

Note: this requires a web server (won't work with `file://` protocol).

## CSS Architecture

All styles are in a single `<style>` block. Organized as:

```
1. CSS Custom Properties (:root)
2. Reset / base styles
3. Typography
4. Layout (nav, sections, footer)
5. Components (cards, tags, buttons)
6. Utilities
7. Responsive overrides (@media)
8. Animations (@keyframes)
9. Accessibility (@prefers-reduced-motion)
10. Dark mode (@prefers-color-scheme: dark) — if implemented
```

## Editing Guidelines

- Always use CSS custom properties for colors, fonts, spacing — never raw values
- Test at 375px, 768px, 1024px, 1440px after any CSS change
- Keep the heading hierarchy clean: one h1, then h2 per section, h3 within sections
- All external links get `target="_blank" rel="noopener noreferrer"`
