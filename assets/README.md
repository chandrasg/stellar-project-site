# Assets Directory

Static files for the site — images, fonts (if self-hosted), and any other media.

## Directory Structure

```
assets/
├── README.md           # This file
├── images/
│   ├── team/           # Team headshots
│   ├── og/             # Open Graph / social sharing images
│   └── screenshots/    # Visual QA screenshots (from Agent 07)
└── fonts/              # Self-hosted fonts (if not using Google Fonts)
```

## Image Specifications

### Team Headshots (`images/team/`)
- **Format**: WebP preferred, JPEG fallback
- **Dimensions**: 400×400px (square, will be displayed smaller)
- **File size**: <100KB per image
- **Naming**: `firstname-lastname.webp` (lowercase, hyphenated)
- **Source**: Official university faculty profile photos ONLY
- **Alt text**: "[Name], [Role] at [Institution]"

### Open Graph Image (`images/og/`)
- **Format**: PNG or JPEG
- **Dimensions**: 1200×630px (standard OG ratio)
- **Content**: Project title, STELLAR logo/mark, institutional logos, Wellcome Trust acknowledgment
- **File size**: <500KB
- **Naming**: `stellar-og.png`

### Screenshots (`images/screenshots/`)
- Generated by Agent 07 during visual QA
- Named by viewport: `mobile-375.png`, `tablet-768.png`, `desktop-1440.png`, `wide-1920.png`
- Not served on the site — for internal QA only

## Font Files (`fonts/`)
Only needed if self-hosting fonts instead of using Google Fonts CDN. Self-hosting is better for performance and privacy but requires:
- WOFF2 format (best compression)
- Proper `@font-face` declarations with `font-display: swap`
- Preload hints in `<head>`

## Rules
- Never commit images larger than 500KB without optimization
- Always provide alt text for every image in the HTML
- Use `loading="lazy"` for images below the fold
- Team photos should be consistent in style (all professional headshots, similar framing)
- Do not use stock photos or AI-generated images for team members
- Social sharing image should be updated whenever the project title or team changes
